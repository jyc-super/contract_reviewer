# index.html만 열어서 실행하는 방안

프로그램 실행을 **사용자가 터미널에 직접 입력하지 않고** 하려는 목적에 맞춰 방안을 정리했습니다.

---

## 1. 왜 index.html 더블클릭만으로는 “전체 프로그램”이 안 되나요?

이 프로젝트의 **실제 앱**은 다음 구조입니다.

- **Next.js** (서버에서 페이지 생성·API 제공)
- **API 라우트** (`/api/contracts`, `/api/contracts/[id]/analyze` 등) → 서버 필수
- **Supabase** (DB), **Gemini** (분석) → 서버에서만 호출

즉, **반드시 Node 서버가 떠 있어야** 업로드·분석·DB 저장이 동작합니다.

- **index.html**은 그런 서버 없이 **정적 목업(프로토타입)** 용입니다.
- **파일 탐색기에서 index.html만 더블클릭**하면 → 그 HTML만 열리고, **전체 기능(업로드·API·DB·분석)은 동작하지 않습니다.**

그래서 “index.html 파일을 열면 모든 프로그램이 실행되게” 하는 것은, **현재 구조(Next + API + DB)를 유지한 채로는 불가능**합니다.  
대신 **“한 번의 더블클릭으로 전체 프로그램이 실행되게”** 하는 방안은 아래와 같이 가능합니다.

---

## 2. 가능한 방안 요약

| 방안 | 사용자 동작 | 결과 |
|------|-------------|------|
| **A. index.html만 열기** | 탐색기에서 `index.html` 더블클릭 | **정적 목업만** 보임 (업로드·API·DB·분석 미동작) |
| **B. 실행기 더블클릭 (권장)** | 프로젝트 폴더에서 **`run.bat`** 더블클릭 | **전체 앱** 실행 (서버 자동 기동 + 브라우저 자동 오픈), 터미널 입력 없음 |
| **C. 정적 빌드 + 별도 백엔드** | Next를 정적 내보내기 후, API는 배포된 서버 사용 | 로컬에서는 “index에 해당하는 페이지”를 연 것처럼 보이지만, API는 여전히 **어딘가의 서버** 필요. 로컬에서 “index만 열면 끝”은 아님 |

---

## 3. 권장: 실행기 더블클릭 (run.bat)

**사용자는 터미널에 아무것도 입력하지 않고**, 프로젝트 폴더에서 **`run.bat`만 더블클릭**하면 됩니다.

- **동작**
  1. PATH를 갱신해 `node`/`npm`을 찾고  
  2. `node_modules`가 없으면 `npm install` 실행 (최초 1회)  
  3. `npm run dev`로 Next 개발 서버 기동  
  4. 몇 초 후 기본 브라우저에서 **http://localhost:3000** 자동 오픈  

- **결과**  
  - **전체 프로그램**(대시보드, 업로드, 계약 상세, 구역 검토, 분석 등)이 그대로 실행됩니다.  
  - “index.html을 열었다”가 아니라 “run.bat을 열었다” 한 번이지만, **사용자 입장에서는 “한 번 더블클릭으로 실행”**입니다.

**파일 위치**: 프로젝트 루트의 **`run.bat`**

---

## 4. index.html만 더블클릭하고 싶을 때

- **원하는 것**: “index.html 파일을 연다”는 한 가지 동작만 하고 싶다.
- **가능한 것**:  
  - **지금처럼** index.html 더블클릭 → **정적 목업만** 보기 (Tailwind CDN으로 UI만 표시, 기능 없음).  
- **불가능한 것**:  
  - index.html만 열어서 **업로드·API·DB·Gemini 분석까지 전부** 동작하게 하는 것은, 현재처럼 Next + API 서버 구조를 쓰는 한 **불가능**합니다. 서버가 꼭 필요하기 때문입니다.

정리하면, **“내가 직접 index.html 파일만 열면 모든 프로그램이 실행되게”** 하는 것은 현재 아키텍처로는 할 수 없고, **“한 번의 더블클릭으로 전체 프로그램이 실행되게”** 하려면 **`run.bat` 더블클릭**을 쓰면 됩니다.

---

## 5. 요약

- **index.html만 열기** → 전체 프로그램 실행 **불가** (정적 목업만 표시).
- **run.bat 더블클릭** → 터미널 입력 없이 **전체 프로그램 실행** (권장).
- `run.bat`은 프로젝트 루트에 있으며, Node.js가 설치·PATH 설정되어 있으면 그대로 사용 가능합니다.
