# run.bat 관리자 모드 실행 테스트 리포트

**테스트 일시**: 2026-03-05  
**실행 환경**: Windows 10, PowerShell/cmd, 관리자 권한(UAC)으로 실행  
**대상**: `run.bat` (환경 설정 단계 포함)

---

## 1. 테스트 목적

- run.bat을 **관리자 모드**로 실행했을 때 **환경 설정**(Windows 방화벽 Node.js 인바운드 허용)이 정상 동작하는지 검증
- 동작 검증: 방화벽 규칙 추가/확인, 개발 서버 기동, 안내 메시지 출력

---

## 2. 실행 방법

- **관리자 권한 실행**: `Start-Process -FilePath 'cmd.exe' -ArgumentList '/c', 'cd /d "d:\coding\contract risk" & run_env_test_admin.bat' -Verb RunAs -Wait`  
  (동일한 방화벽 확인/추가 로직을 사용하는 테스트용 배치로 검증)
- **일반 실행**: 프로젝트 폴더에서 `run.bat` 실행하여 전체 흐름(환경 설정 메시지 + npm run dev) 확인

---

## 3. run.bat 실행 출력 요약

| 단계 | 출력/동작 |
|------|-----------|
| 환경 설정 | `[환경 설정] Node.js 방화벽 규칙이 이미 있습니다.` (규칙이 있는 경우) |
| 의존성 | `node_modules\next` 존재 시 npm install 생략, `up to date, audited 192 packages` |
| 서버 안내 | `개발 서버 시작 중... 잠시 후 브라우저가 열립니다.` |
| | `포트 3000이 사용 중이면 터미널에 표시된 주소(예: http://localhost:3001)로 접속하세요.` |
| | `접속이 안 되면 docs\TEST_RUNNER_REPORT_TIMEOUT.md 의 체크리스트를 참고하세요.` |
| 서버 기동 | `next dev -H 0.0.0.0 -p 3000` 실행, Next.js 개발 서버 리스닝 |

---

## 4. 검증 결과

### 4.1 방화벽 규칙

- **확인 명령**: `netsh advfirewall firewall show rule name="Node.js"`
- **결과**: 규칙 존재함. 사용(예), 방향(들어오는 메시지), 프로필(도메인, 개인, 공용), 작업(허용).
- **해석**: 관리자 모드로 run.bat(또는 동일 로직 테스트 배치) 실행 시 방화벽 규칙이 추가되었거나, 이전에 추가된 규칙이 유지되고 있음.

### 4.2 환경 설정 단계

- run.bat 상단의 **환경 설정** 블록이 실행되어 다음 중 하나가 출력됨:
  - 규칙 없음 → 추가 시도 → 성공 시: `[환경 설정] Node.js 방화벽 규칙을 추가했습니다.`
  - 규칙 없음 → 추가 실패(비관리자): `[안내] 방화벽 규칙 추가에 실패했습니다. ... docs\TEST_RUNNER_REPORT_TIMEOUT.md 를 참고하세요.`
  - 규칙 있음: `[환경 설정] Node.js 방화벽 규칙이 이미 있습니다.`
- 실제 실행 시 **규칙이 이미 있음** 메시지 확인됨.

### 4.3 개발 서버 기동

- `npm run dev` 정상 실행, `next dev -H 0.0.0.0 -p 3000` 기동 확인.
- 기존 [RUN_BAT_VERIFICATION.md](RUN_BAT_VERIFICATION.md)와 동일하게 Next.js 개발 서버가 정상 기동하는 흐름 유지.

### 4.4 접속 안내

- 포트 3000 사용 시 대체 포트 안내 및 `docs\TEST_RUNNER_REPORT_TIMEOUT.md` 체크리스트 안내 문구가 출력됨.

## 6. 권장 사용 방법

- **최초 실행 또는 localhost 접속이 안 될 때**: run.bat을 **우클릭 → 관리자 권한으로 실행**하여 방화벽 규칙 추가 후, 이후부터는 일반 실행으로 사용.
- **일반 사용**: 프로젝트 폴더에서 run.bat 더블클릭 또는 터미널에서 `run.bat` 실행.
- 접속 문제 시: [TEST_RUNNER_REPORT_TIMEOUT.md](TEST_RUNNER_REPORT_TIMEOUT.md) 체크리스트 참고.
